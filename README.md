# Weiyu Gateway for Home Assistant（非官方）

![Weiyu Logo](./custom_components/weiyu_gateway/logo.png)

在**不依赖原厂云**的前提下，让 Home Assistant 与微羽类智能网关、子设备（断路器 / 漏电保护等）在局域网内**直连**：发现网关、注册回调地址、接收上报、下发控制。

**当前集成版本：1.2.14**（与 `manifest.json` 中 `version` 一致）

---

## 为什么做这个项目

部分网关与终端硬件仍可用，但官方平台或 App 已不可用或不便使用。  
本集成把控制权留在本地：发现、连接、监控、告警与合闸分闸均在 HA 内完成。

若对你有帮助，欢迎 Star、提 Issue、一起完善。

---

## 功能概览

- **本地直连**：UDP 发现网关并注册 HA 监听地址，网关主动回连（`iot_class`: `local_push`）
- **子设备**：断路器开关、网关**总开关**（一键合闸/分闸，受子设备手动模式保护）
- **电参量传感器**：电压、电流、有功功率、电能、频率、功率因数、火线/芯片温度；2P 设备支持**零线温度**；漏保支持**漏电流**与**漏电自检**按钮
- **运行状态**：中文合并状态（含离线、断电、锁定、告警/故障细分、脱扣/预脱扣等）；手动模式在状态中体现为**锁定**（已移除单独的「工作模式」实体）
- **告警**：进入异常时持久通知 + 事件总线；支持附带故障/报警记录快照（有字段时）；**忽略协议中保留/未定义位**，减少误报
- **连接健壮性**：重连后子设备扫描、会话代数避免旧 TCP 读循环误标离线、重注册串行化、静默超时与读任务清理等
- **时间同步**：网关每次 TCP 连上后向网关校准时间（若固件支持）
- **中文化**：配置流程与实体命名（`translations/zh-Hans.json`）

## 适用场景

- 希望减少云依赖、局域网内自控
- 需要将断路器 / 漏保状态与电参接入 Home Assistant 自动化或能源面板

## 安装

### 手动安装

将目录 `custom_components/weiyu_gateway` 复制到 Home Assistant 配置目录：

`/config/custom_components/`

重启 Home Assistant。

### HACS 自定义仓库

仓库根目录包含 `hacs.json`，可作为 **HACS → 自定义存储库** 添加，集成名称：**Weiyu Gateway**。

---

## 快速开始

1. **设置 → 设备与服务 → 添加集成**，搜索 **Weiyu Gateway**
2. 推荐选择 **「自动搜索网关」**，在列表中选网关 IP
3. 填写 **Home Assistant 可被网关访问的 `listen_ip`** 与 **监听端口**（需与防火墙放行一致）
4. 保存后在 **设备** 页查看网关与子设备实体

### 网络与端口提示

- 网关侧使用 **UDP 50500** 完成发现与注册；网关会按注册信息**反连** HA 的 `listen_ip:listen_port`
- HA 主机防火墙需放行上述监听端口；HA 与网关需在同一可达二层/路由环境（跨 VLAN 时注意广播与单播策略）

## 数据与行为说明

- **以上报为主**：子设备遥测随网关 `subclass` 推送更新；若某一帧缺少部分字段，会**沿用上一帧有效值**，减少界面出现「未知」
- **静默检测**：长时间无任何协议帧时，会将网关标为离线并尝试重注册（阈值与网关上报周期相关，并留有裕量）
- 不同型号子设备字段可能略有差异，以实际固件上报为准
